## 프로젝트 결정의 진화

1. **초기**: 외부 GLTF/OBJ 모델 사용 시도
   - 문제: 단일 메시로 관절 제어 불가능
   - 해결: 프로그래밍 방식으로 로봇 생성

2. **모델 구조**: Three.js Group 계층 구조 사용
   - 각 관절을 Group으로 생성
   - 부모-자식 관계로 정확한 회전 구현

3. **관절 방향**: X축 회전 관절은 원통을 눕힘
   - Shoulder, Elbow, Wrist2는 X축 방향으로 원통 배치
   - 회전 중심을 관절 중심으로 설정

4. **UI 레이아웃**: 반응형 디자인
   - 데스크톱: 좌우 분할 (3D 뷰 + 슬라이더 패널)
   - 모바일: 상하 분할 (3D 뷰 60% + 슬라이더 40%)

5. **프로젝트 방향**: 범용 데모로 변경
   - 처음: 레인보우로보틱스 면접 포트폴리오
   - 최종: Three.js Demo (범용 데모)

6. **파일 정리**: 불필요한 파일 제거
   - 외부 모델 파일 삭제
   - 사용하지 않는 유틸리티 삭제
   - 문서 정리
